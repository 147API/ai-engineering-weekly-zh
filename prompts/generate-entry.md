### Prompt：生成资源条目（中文）

用途：给你一个 GitHub 仓库链接，让 AI 生成符合本仓库格式的“资源条目草稿”，你只需复核与补引用。

#### 输入给 AI 的内容（你填空）

```text
你是资深 AI 工程实践编辑。请为下面这个 GitHub 仓库生成“资源条目草稿”，用于中文 AI 工程资源导航。

仓库链接：<粘贴 GitHub URL>
建议分类：<从下列分类中选 1 个：agents/workflows | RAG/检索/向量数据库 | evals/testing | 可观测/调试/streaming | 推理/部署/runtime | 安全/合规/脱敏 | 文档/数据处理 | 开发者工具>

硬性要求：
1) 输出必须严格使用我们条目模板（Markdown，带项目/一句话价值/适用场景/注意事项/标签/引用）
2) 禁止使用营销措辞（最强、吊打、必备等）
3) “注意事项/不适用”至少 1 条，越具体越好
4) “引用/参考”至少 1 条：优先给出该仓库 README 或 Docs 的链接（如果你无法确认 Docs 链接，请只给仓库链接并标注 TODO）
5) 如果你不确定任何事实（例如功能细节、性能数据、兼容性），请用“可能/需要人工确认”并标注 TODO，不要编造
```

#### 期望输出格式（AI 必须照抄结构）

请输出：

```md
- **项目**：[owner/repo](https://github.com/owner/repo)
  - **一句话价值**：
  - **适用场景**：
    - 
  - **注意事项 / 不适用**：
    - 
  - **标签**：``
  - **引用/参考**：
    - README/Docs：
```
