### 1. 本周精选

- **项目**：[LMCache/LMCache](https://github.com/LMCache/LMCache)
  - **一句话价值**：面向 LLM Serving（如 vLLM）的 KV Cache 共享层，通过在实例间复用 KV 缓存显著降低首字延迟（TTFT）。
  - **适用场景**：多轮对话、长文档 RAG、分布式推理场景下的 Context Caching 优化。
  - **注意事项**：需配合 vLLM 等后端使用，部署时涉及存储层配置（支持 CPU/Disk/S3）。
  - **引用**：https://github.com/LMCache/LMCache

- **项目**：[alibaba/OpenSandbox](https://github.com/alibaba/OpenSandbox)
  - **一句话价值**：通用的 AI 应用沙箱平台，支持 Docker/K8s 运行时，为 Coding Agent 和 GUI Agent 提供安全的执行环境。
  - **适用场景**：构建代码生成 Agent、评测 Agent 安全性、需要隔离运行不可信代码的场景。
  - **注意事项**：生产环境建议配合 gVisor 或 Kata Containers 等安全容器运行时以增强隔离性。
  - **引用**：https://github.com/alibaba/OpenSandbox

- **项目**：[bytedance/deer-flow](https://github.com/bytedance/deer-flow)
  - **一句话价值**：字节跳动开源的 Super Agent 框架（原 Deep Research 升级版），支持子 Agent 编排、沙箱执行与长期记忆。
  - **适用场景**：构建处理复杂、长流程任务（如深度调研、自动化编码）的 Agentic 应用。
  - **注意事项**：2.0 版本重构后专注于 Agent Harness（工程脚手架），上手门槛较 1.x 略高，但扩展性更强。
  - **引用**：https://github.com/bytedance/deer-flow

- **项目**：[microsoft/markitdown](https://github.com/microsoft/markitdown)
  - **一句话价值**：微软开源的文档转换工具，能将 PDF、PowerPoint、Word、Excel 等格式高保真转换为 Markdown。
  - **适用场景**：RAG 数据清洗管道、将非结构化文档转换为 LLM 友好的上下文格式。
  - **注意事项**：对复杂排版（如多栏 PDF）的处理能力仍受限于底层解析库，建议结合 OCR 增强。
  - **引用**：https://github.com/microsoft/markitdown

- **项目**：[anthropics/claude-code-security-review](https://github.com/anthropics/claude-code-security-review)
  - **一句话价值**：基于 Claude 模型的 GitHub Action，专注于对 Pull Request 进行语义级的安全漏洞审计。
  - **适用场景**：CI/CD 流水线中的自动化代码审查、辅助人工 Security Review。
  - **注意事项**：依赖 Claude API，需注意 Token 消耗成本；建议仅对敏感目录或特定分支开启。
  - **引用**：https://github.com/anthropics/claude-code-security-review

- **项目**：[modelscope/FunASR](https://github.com/modelscope/FunASR)
  - **一句话价值**：阿里达摩院开源的基础语音识别工具包，包含 SenseVoice、Paraformer 等工业级模型。
  - **适用场景**：搭建私有化语音转文字（STT）服务、实时语音听写、说话人分离（Diarization）。
  - **注意事项**：提供 Runtime SDK 方便部署，但对显存/CPU 资源有一定要求，需根据并发量压测。
  - **引用**：https://github.com/modelscope/FunASR

- **项目**：[adap/flower](https://github.com/adap/flower)
  - **一句话价值**：友好的联邦学习（Federated Learning）框架，支持在分布式设备（手机、IoT、边缘节点）上协同训练模型而不共享数据。
  - **适用场景**：医疗、金融等数据隐私敏感场景，或利用边缘设备算力进行模型微调。
  - **注意事项**：联邦学习涉及复杂的通信与聚合策略，调试难度高于集中式训练。
  - **引用**：https://github.com/adap/flower

- **项目**：[open-edge-platform/anomalib](https://github.com/open-edge-platform/anomalib)
  - **一句话价值**：基于 PyTorch Lightning 的深度学习异常检测库，支持从训练到 OpenVINO 边缘部署的全流程。
  - **适用场景**：工业质检（缺陷检测）、医疗影像异常识别、视频监控异常行为检测。
  - **注意事项**：专注于“视觉”异常检测，非时序/日志类异常检测工具；提供了好用的无代码 GUI（Anomalib Studio）。
  - **引用**：https://github.com/open-edge-platform/anomalib
