### 1. 本周精选（10）

- **项目**：[langchain-ai/langgraph](https://github.com/langchain-ai/langgraph)
  - **一句话价值**：用于构建“有状态的 Agent/工作流”的图编排框架，适合把多步推理与工具调用做成可控流程。
  - **适用场景**：复杂任务拆解、需要明确状态/分支/循环的 agentic workflow。
  - **注意事项**：流程越复杂越需要评测与可观测；建议先从最小图跑通再扩展。
  - **引用**：https://github.com/langchain-ai/langgraph

- **项目**：[microsoft/semantic-kernel](https://github.com/microsoft/semantic-kernel)
  - **一句话价值**：面向 LLM 应用的 SDK/编排框架，强调与企业工程体系集成（工具调用、规划、连接器等）。
  - **适用场景**：企业级集成、需要在既有系统中引入 LLM 编排与工具调用。
  - **注意事项**：框架能力多，上手时建议先锁定一个关键用例，避免“全都用上导致复杂度爆炸”。
  - **引用**：https://github.com/microsoft/semantic-kernel

- **项目**：[weaviate/weaviate](https://github.com/weaviate/weaviate)
  - **一句话价值**：开源向量数据库/检索平台，支持向量检索与过滤，适合做 RAG 的检索底座。
  - **适用场景**：自托管向量检索、需要 metadata filter 的检索增强应用。
  - **注意事项**：向量库只是底座，RAG 质量仍需靠数据清洗、chunk 策略与评测回归。
  - **引用**：https://github.com/weaviate/weaviate

- **项目**：[comet-ml/opik](https://github.com/comet-ml/opik)
  - **一句话价值**：开源的 LLM 应用调试/评测/监控平台，提供追踪、自动化评测与仪表盘。
  - **适用场景**：把 LLM 应用“跑出来以后”做可观测、可评测、可对比。
  - **注意事项**：平台型工具需要采集/存储/权限体系配合；先从最核心链路接入更稳。
  - **引用**：https://github.com/comet-ml/opik

- **项目**：[lunary-ai/lunary](https://github.com/lunary-ai/lunary)
  - **一句话价值**：LLM 生产工具箱（可观测、prompt 管理与评测），偏“面向落地”的一体化平台。
  - **适用场景**：线上追踪 LLM 调用、管理 prompt 版本、做实验对比。
  - **注意事项**：接入前要明确脱敏/合规策略，避免把敏感数据无脑上报。
  - **引用**：https://github.com/lunary-ai/lunary

- **项目**：[huggingface/text-generation-inference](https://github.com/huggingface/text-generation-inference)
  - **一句话价值**：Hugging Face 的推理服务（TGI），用于高性能文本生成模型 serving。
  - **适用场景**：自建推理服务、需要较成熟的 serving 组件与部署经验。
  - **注意事项**：部署与性能调优涉及系统与模型细节；建议先在小规模压测后再上生产。
  - **引用**：https://github.com/huggingface/text-generation-inference

- **项目**：[NVIDIA/TensorRT-LLM](https://github.com/NVIDIA/TensorRT-LLM)
  - **一句话价值**：面向 NVIDIA 生态的高性能 LLM 推理优化与部署工具链。
  - **适用场景**：推理瓶颈明显、需要更极致性能/吞吐的部署场景。
  - **注意事项**：上手门槛偏系统与工程化（CUDA/驱动/模型格式），评估团队能力与维护成本。
  - **引用**：https://github.com/NVIDIA/TensorRT-LLM

- **项目**：[Azure/PyRIT](https://github.com/Azure/PyRIT)
  - **一句话价值**：面向 LLM 的安全测试/红队工具（提示词注入、越权等攻击面），帮助建立安全评测与回归用例。
  - **适用场景**：做 LLM 应用的安全评审、红队测试、上线前安全回归。
  - **注意事项**：安全测试结果需要落到工程措施（权限隔离、最小权限、审计、guardrails），别停留在“跑报告”。
  - **引用**：https://github.com/Azure/PyRIT

- **项目**：[docling-project/docling](https://github.com/docling-project/docling)
  - **一句话价值**：把 PDF/DOCX/HTML 等文档解析成统一结构，面向“文档进入 GenAI/RAG”场景的工程化 SDK/CLI。
  - **适用场景**：RAG 数据前处理、文档抽取与结构化、文档到知识库流水线。
  - **注意事项**：文档解析质量需针对自家数据做抽样评测与回归（尤其表格/复杂排版）。
  - **引用**：https://github.com/docling-project/docling

- **项目**：[anomalyco/opencode](https://github.com/anomalyco/opencode)
  - **一句话价值**：开源 AI 编程/代码代理工具，强调在本地代码库上下文中完成任务。
  - **适用场景**：个人/小团队用 AI 加速开发（重构、脚手架、批量修改）。
  - **注意事项**：仍需严格 code review 与测试；建议限制写权限范围与敏感文件访问。
  - **引用**：https://github.com/anomalyco/opencode
